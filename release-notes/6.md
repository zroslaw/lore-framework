# Release Notes — Version 6

## Summary

Version 6 makes agent boot faster by replacing the unresolved `${CLAUDE_PLUGIN_ROOT}` variable in shortcut commands with hardcoded absolute paths. Boot via a shortcut command now skips the agent discovery scan entirely.

**Migration required:** yes — migration 6 regenerates shortcut commands in `.claude/commands/` with absolute paths.

## What's New

### Shortcut commands with absolute paths

Shortcut commands (`.claude/commands/lr-<agent-name>-agent.md`) previously contained:

```
Read `${CLAUDE_PLUGIN_ROOT}/docs/agent-boot.md` and boot as agent `<agent-name>`.
```

The problem: Claude Code does not resolve `${CLAUDE_PLUGIN_ROOT}` in custom commands (only in plugin skill files). The model received the literal `${CLAUDE_PLUGIN_ROOT}` string and had to search for the plugin directory at runtime — wasting multiple tool calls on path discovery.

The new format:

```
Read `<absolute-path>/docs/agent-boot.md` and boot as agent `<agent-name>` from `<absolute-agent-dir>`.
```

Both paths are resolved at registration time (when `/lr:register-repo` runs) and hardcoded. If the plugin or agent repo moves, re-run `/lr:register-repo` to regenerate.

### Boot skips discovery when path is provided

`agent-boot.md` step 1 now accepts an optional absolute path to the agent directory. When provided (as shortcut commands now do), boot uses it directly — verifying `role.md` exists and deriving the repo root — instead of scanning all directories for lore agent repos. This saves one Glob call and eliminates the discovery scan entirely.

The generic `/lr:boot <agent-name>` skill still performs full discovery (no path provided), so it remains the fallback when the agent location is unknown.

### Terminology: "shortcut commands"

What was previously called "per-agent boot commands" or "registered agent commands" is now **shortcut commands**. The name reflects their purpose: convenience shortcuts that delegate to `agent-boot.md` with pre-resolved paths. The main boot logic lives in the `/lr:boot` skill and `agent-boot.md`.

### Updated `/lr:check` validations

Check 18 now detects both legacy shortcut command formats:
- `lore-framework/docs/agent-boot.md` (pre-v5 sibling-path)
- `${CLAUDE_PLUGIN_ROOT}/docs/agent-boot.md` (v5, unresolved in custom commands)

Both are flagged with a suggestion to run `/lr:update` or `/lr:register-repo`.

## What Hasn't Changed

- No changes to lore topics, `lore-context.md`, `role.md`, `lore/`, or `workdir/` contents
- `/lr:boot <agent-name>` still works via full discovery (no migration needed for generic boot)
- Attach / consult / recall / finalization behavior is unchanged
- All v4/v5 collaboration and migration mechanics are preserved

## Impact

- **Re-registration needed after plugin moves.** Since paths are absolute, moving the plugin installation or the agent repo invalidates shortcut commands. Run `/lr:register-repo <lore-agent-repo>` to regenerate. This is the trade-off for faster boot.
- **Repos at v4 or earlier** will run migrations 5 and 6 sequentially on next boot or `/lr:update`. Migration 5 handles orphan cleanup; migration 6 handles the path format upgrade.
