# Migration to Version 6

## Summary

Regenerates shortcut commands (`.claude/commands/lr-*-agent.md`) to use absolute paths instead of `${CLAUDE_PLUGIN_ROOT}`. This eliminates the boot-time discovery scan and the unresolved `${CLAUDE_PLUGIN_ROOT}` variable (which Claude Code does not resolve in custom commands).

**Changes:**
1. **Regenerate shortcut commands** with absolute paths to both `agent-boot.md` and the agent directory.

## Scope

Apply per lore agent repo. The file modifications happen in `.claude/commands/` in the **domain directory**. The first repo to reach v6 applies it; subsequent repos re-run idempotently and find nothing to do.

## Steps

All steps must be idempotent — safe to re-run. Do **not** stamp the new version in `lore-repo.md` within this migration; the update process handles that after all steps succeed.

### Step 1: Discover agents

Scan all directories in the domain for lore agent repos (directories containing `lore-repo.md` at the root). For each repo, scan `agents/` for agent directories (containing `role.md`). Build a mapping of agent-name to absolute agent directory path.

### Step 2: Resolve agent-boot.md path

Determine the absolute path to `docs/agent-boot.md` within the plugin. Derive it from the path you used to read this migration file — go up one level from `migrations/` to the plugin root, then into `docs/agent-boot.md`. Call this `<agent-boot-path>`.

### Step 3: Regenerate shortcut commands

Scan `<domain>/.claude/commands/` for files matching `lr-*-agent.md`. For each one:

1. Read the file content.
2. Determine whether it is a shortcut command (content contains the phrase `boot as agent`). If not, skip.
3. Extract the agent name from the filename: strip the `lr-` prefix and the `-agent.md` suffix.
4. Look up the agent name in the mapping from Step 1. If the agent is not found in any repo, **warn** and skip — the shortcut command points to an agent that no longer exists.
5. Compute the new shortcut command content — exactly this single line plus a trailing newline:

   ```
   Read `<agent-boot-path>` and boot as agent `<agent-name>` from `<agent-dir>`.
   ```

   Where `<agent-boot-path>` is from Step 2, `<agent-name>` is the extracted name, and `<agent-dir>` is the absolute path from the mapping.

6. **If the on-disk content matches the new content exactly**, skip (idempotent — already migrated).
7. **If the on-disk content matches the known v5 template** (see below), overwrite with the new content.
8. **If the on-disk content diverges from both**, treat as manually edited. Follow the manual-edit handling described in `update.md` — present a three-way merge (old template to new template onto on-disk content) and proceed per the user's choice.

Record each regenerated file for the final report.

#### Known clean v5 template

Exactly this single line (plus a trailing newline):

```
Read `${CLAUDE_PLUGIN_ROOT}/docs/agent-boot.md` and boot as agent `<agent-name>`.
```

A file matches when the template equals the on-disk content exactly (modulo trailing whitespace), with `<agent-name>` substituted by the kebab-case name extracted from the filename.

#### Known clean pre-v5 templates

A file also matches as "clean" (safely overwritable) if it matches any of these older forms:

```
Read `lore-framework/docs/agent-boot.md` and boot as agent `<agent-name>`.
```

This covers repos that jumped directly from v4 or earlier to v6 without going through v5 first.

### Step 4: Report

After all commands are processed, print a summary:

- **Regenerated:** list of `lr-<agent-name>-agent.md` files rewritten (or "none").
- **Merged manually:** list of files that needed three-way merge (or "none").
- **Skipped (missing agent):** list of commands whose agent was not found (or "none").

## Dry-Run Mode

When invoked from `/lr:update --dry-run`, detect all the same cases but do not modify any files. Report each would-be change:

- `would regenerate: <path>`
- `would prompt (manual edits): <path>`
- `would warn (missing agent): <path>`

Do not prompt the user in dry-run mode.

## Notes

- All steps in this migration are idempotent — safe to re-run after a partial failure.
- `.claude/commands/` lives in the **domain directory**, shared across all lore agent repos. Running this migration against the first repo handles all shared commands; subsequent repos find nothing to do.
- Do not update `lore-repo.md` frontmatter in this doc. The `/lr:update` process stamps the repo version only after all migrations succeed.
